## Por que estes roadmaps?

A maioria dos roadmaps na internet é uma lista de tecnologias sem contexto. Estes são diferentes:

- **Rastreáveis** — cada item é um checkbox que salva seu progresso no localStorage
- **Projetos reais** — cada fase tem um projeto com entregáveis claros e dicas de como desenvolver
- **Opiniões e trade-offs** — não é "aprenda tudo", é "aprenda isso por esse motivo"
- **Referências com links** — livros, documentações oficiais, cursos e ferramentas para cada tópico
- **Um arquivo HTML** — abra no browser, sem servidor, sem build, sem conta

## 🧰 Como Usar

### Opção 1 — GitHub Pages (recomendado)

1. Faça fork deste repositório
2. Ative GitHub Pages em **Settings → Pages → Source: main branch**
3. Acesse em `https://seuusuario.github.io/roadmaps/`

### Opção 2 — Local

```bash
git clone https://github.com/luanpanno/roadmaps.git
open roadmaps/index.html     # macOS
xdg-open roadmaps/index.html # Linux
start roadmaps/index.html    # Windows
```

### Progresso

- Clique nos checkboxes para marcar itens como concluídos
- O progresso é salvo automaticamente no **localStorage** do seu browser
- A tab ativa e as seções expandidas também são persistidas
- Para resetar o progresso, limpe o localStorage do site

## 🎯 Para Quem é Isso?

**Full Stack Sênior** é para você se:

- Trabalha com Node.js/React/TypeScript há 2+ anos
- Sabe fazer funcionar, mas quer aprender a fazer **bem**
- Quer se preparar para entrevistas de sênior
- Sente que falta base em arquitetura, infra ou banco de dados

**Low-Level** é para você se:

- Quer entender o que acontece quando `fetch()` é chamado — de verdade
- Tem curiosidade sobre C, memória, como o SO funciona
- Quer complementar seu conhecimento web com fundamentos de sistemas
- Está disposto a investir tempo em profundidade, não em hype
